# SSTV фигня. 
Тут есть немного документации, чужой sstv-codec (источник - посмотрите, откуда форкнуто) и колхозный скрипт для очень конкретного юзкейса который никому наверное не нужен будет.
 
Итоговым "результатом" выполнения должно быть видео с кривым эффектом старого тв.
# Что нужно
 - numpy и Pillow
 - ваше видео
 - ffmpeg (при желании можно заменить на альтернативу, если такая есть)
 - несколько часов свободного времени
# Таки сам гайд
Сначала вам нужно узнать несколько вещей про ваше видео. Для этого пишете
```
ffprobe [ваше видео]
```
и ищете строчку, похожую на
```
Stream #0:0[0x1](und): Video: h264 (High) (avc1 / 0x31637661), yuvj420p(pc, bt470bg/unknown/unknown, progressive), 1920x880 [SAR 55:96 DAR 5:4], 5083 kb/s, 15 fps, 15 tbr, 15360 tbn (default) 
```
ключевое слово - ***похожую***, она может во многом отличаться. Отсюда нам нужен fps (в данном случае 15) и разрешение (в данном случае 1920x880). Дальше желательно прикинуть, сколько кадров в видео - точное число не нужно, нужно максимальное число цифр. Для этого продолжительность видео в секундах помножьте на калькуляторе на fps. После этого разбиваем видео на отдельные кадры:
```
ffmpeg -i [ваше видео] %0[количество нулей[d.jpg
```
Поздравляем, теперь у вас вся папка в кадрах. Теперь посмотрите, какой самый последний кадр по номеру (у какого фото в названии самое большое число). Дальше идёте в maketv.sh.
Первая строчка там выглядит примерно так:
```
for file in {002..418}.jpg; do
```
{002..418}.jpg это в переводе с баша все файлы от 002.jpg до 418.jpg. Соотвественно вам надо от вашего первого кадра до вашего последнего кадра. 
Например, если первый кадр это 00001.jpg, а последний 99999.jpg, то будет {00001..99999}.jpg.

Когда поменяете это запускаете maketv.sh в терминале и расслабляетесь, пьёте чашечку кофе или чая, потому что займёт всё дело несколько часов.

Когда скрипт завершиться вам нужно заново собрать видео по кадрам. Для этого
```
ffmpeg -framerate [ваш fps] -i %0[количество нулей]d.jpg -vf "scale=ваше разрешение" видео.mp4
```
Получиться видео без звука. Звук получается примерно так
```
ffmpeg -i [оригинальное видео] звук.mp3
ffmpeg -i видео.mp4 -i звук.mp3 видео_итоговое.mp4
```
Вот, собственно, и всё
# Бонусы
в скрипте вместо robot72 можно поставить другой режим, и результат будет другим. А ещё можно кодировать и декодировать в разных форматах, тогда вообще каша получается.

Чтобы наложить логотп тв-канала сделайте в gimp файл с таким же разрешением, что и видео, сделайте его прозрачным, вставтье в уголок логотип, сохраните и сделайте
```
ffmpeg -i vid.mp4 -i logo.png  -filter_complex "[0:v][1:v]overlay=10:10" -c:a copy end.mp4
```
(-c:a copy вылетит, если нет аудиодорожки)
